# SecureChat

kubell様へ面接に向けて現在開発を継続中です。現時点での実装内容を提出しております。

1.制作背景(課題・目的)
　従来のchat アプリは履歴が残るため機密情報(例えばコード)等を載せることはできなかっ
た。しかし、本アプリは選択したユーザーのみが閲覧できて、閲覧したら確認メールが投稿
主に送られてきて、そして本投稿を完全に削除することができる。
2.技術選定及びその意図
　SpringBoot: DB操作やハッシュ値にするのがアノテーション一と設定ちょっとでできて楽で、さらにセキュリティを扱うことから実際運用するなら大人数想定。よって保守性高めのSpringBootを選定。あと速度等のパフォーマンスにはあまりこだわる必要が無いのも選定理由の一つ。
　PostgreSQL: 顧客データ扱ってセキュリティに注意しないといけないから厳密なPostgreSQLを選定。
　gradle:後にモバイルアプリへの拡張を考えて選定
